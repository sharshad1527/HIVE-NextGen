# **Contributing to H.I.V.E. NextGen**

First off, thank you for considering contributing to **H.I.V.E. NextGen**\! It’s people like you who make open-source tools better than paid, locked-down alternatives.  
By contributing, you help build a world where creators don't have to pay a subscription just to use a glitch effect.

## **🛠️ How Can I Contribute?**

### **1\. Reporting Bugs**

If you find a bug, please open an **Issue** on GitHub. Include:

* A clear, descriptive title.  
* Steps to reproduce the bug.  
* What you expected to happen vs. what actually happened.  
* Your operating system and Python version.

### **2\. Suggesting Features**

Have an idea for a new automation tool or a cool UI tweak?

* Open an Issue with the tag enhancement.  
* Explain why this feature would be useful for creators.

### **3\. Adding New Presets (Easy Entry\!)**

H.I.V.E. is designed to be extensible. You can contribute new visual styles without writing complex code:

* Navigate to the presets/ folder.  
* Create new JSON files for **Captions**, **Effects**, or **Transitions**.  
* Test them in the app and submit a PR\!

### **4\. Code Contributions**

If you want to dive into the core engine:

1. **Fork** the repository.  
2. **Clone** your fork: git clone https://github.com/sharshad1527/HIVE\_NextGen.git.  
3. Create a **Feature Branch**: git checkout \-b feature/AmazingFeature.  
4. **Commit** your changes: git commit \-m 'Add some AmazingFeature'.  
5. **Push** to the branch: git push origin feature/AmazingFeature.  
6. Open a **Pull Request**.

## **📜 Code Style Guidelines**

* **Python**: We follow [PEP 8](https://www.python.org/dev/peps/pep-0008/) standards. Keep your code clean and readable.  
* **UI (PySide6)**: Ensure new UI elements follow the dark-themed, minimalist aesthetic defined in styles/theme.qss.  
* **Documentation**: If you add a new core function, please include a docstring explaining what it does.  
* **Architecture**: Respect the separation between the core/ (logic/rendering) and ui/ (interface) folders.

## **⚖️ License & Ownership**

This ensures that the project remains free and open for everyone, forever. No one—including us—can ever take your contributions and turn them into a paid, closed-source product.

## **💬 Communication**

If you have questions or want to brainstorm a big feature before coding it, feel free to start a **Discussion** on the GitHub repository.
